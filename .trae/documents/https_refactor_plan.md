# HTTPS 服务重构与集成实现计划

## 1. 项目现状分析

### 1.1 现有结构

* **CentralBoard/**: 中央控制板代码

  * `Central.go`: 主服务文件

  * `config.yaml`: 配置文件

  * `MavlinkCommand/`: MAVLink 通信模块

  * `Distribute/`: 无人机搜索模块

  * `Handler/`: 处理器模块

  * `Shared/`: 共享数据结构

### 1.2 向后端传输消息-规范

* **后端的服务地址**: <https://api.deeppluse.dpdns.org>

* **认证方式**: JWT Token (Bearer Token)

* **核心接口**: POST /api/board/send-message

* **消息格式**: 严格按照 API 文档规范

## 3. 实现计划

### 3.1 阶段一：项目结构重构

#### 3.1.1 创建目录结构

```
CentralBoard/
├── app/
│   ├── api/           # 路由处理
│   │   ├── handlers/  # 请求处理器
│   │   └── routes.go  # 路由注册
│   ├── services/      # 业务逻辑
│   │   ├── backend/   # 后端通信服务
│   │   ├── mavlink/    # MAVLink 服务
│   │   └── task/       # 任务处理服务
│   ├── config/         # 配置管理
│   └── utils/          # 工具函数
├── config.yaml         # 主配置文件
├── Central.go          # 主入口
├── docs/               # 文档
│   ├── README.md
│   └── API.md
└── tests/              # 测试程序
    ├── integration/
    └── unit/
```

#### 3.1.2 核心文件拆分

* **Central.go**: 主入口，初始化服务

* **app/api/routes.go**: 路由注册

* **app/api/handlers/**: 按功能分组的处理器

* **app/services/backend/**: 后端 API 通信

* **app/services/mavlink/**: MAVLink 命令处理

* **app/services/task/**: 任务链管理

### 3.2 阶段二：HTTPS 服务实现

#### 3.2.1 配置管理

* 扩展 `config.yaml` 支持 HTTPS 配置

* 支持 Let's Encrypt 自动证书

* 支持手动证书配置

#### 3.2.2 路由系统

* 使用 Gin 框架实现 RESTful API

* 路由分组：`/api/board`、`/api/health` 等

* 中间件：CORS、日志、错误处理

#### 3.2.3 后端通信

* 实现 `app/services/backend/client.go`

* 支持设备认证（JWT Token）

* 实现 POST /api/board/send-message 接口

* 严格按照 API 文档格式传递消息

### 3.3 阶段三：业务逻辑实现
